## 📖 如何使用

### 第一步：修改个人信息
1. 打开 `README.md` 查看详细说明
2. 按照注释修改每个 HTML 文件
3. 重点修改的内容：
   - 你的名字
   - 职位/身份
   - 联系方式
   - 个人简介
   - 社交媒体链接

### 第二步：替换个人照片
1. 准备一张个人照片（推荐 400x400 像素）
2. 命名为 `profile.png` 或 `profile.jpg`
3. 放入 `images/` 文件夹
4. 如果使用 .jpg，记得修改 HTML 中的扩展名

### 第三步：添加内容
1. 添加你的研究论文
2. 添加演讲记录
3. 更新活动参与
4. 添加教学经历
5. 上传相关文件（PDF、幻灯片等）

### 第四步：部署到 GitHub Pages
1. 打开 `DEPLOYMENT.md` 查看详细步骤
2. 确保仓库名是 `your-username.github.io`
3. 启用 GitHub Pages
4. 等待几分钟后访问你的网站

## 🔍 注意事项

### 必须修改的地方
以下是你**必须**修改的内容（搜索这些关键词）：
- `你的名字`
- `Your Name`
- `你的职位或身份`
- `your-username`
- `your-github-username`
- `your.email@example.com`
- 所有带有 `[方括号]` 的占位符内容

### 可选修改的地方
- 社交媒体图标（可以删除不需要的）
- 页面（可以删除不需要的页面）
- 导航栏（可以调整顺序或删除项目）
- 颜色和样式（修改 CSS 文件）

### 不要修改的地方
- CSS 和 JavaScript 文件（除非你知道在做什么）
- Bootstrap 相关的类名
- 文件夹结构

你可以通过以下方式预览网站：

## 🆘 如果遇到问题

1. **查看文档**
   - README.md - 使用说明
   - DEPLOYMENT.md - 部署指南

2. **检查清单**
   - 是否所有占位符都已替换？
   - 文件路径是否正确？
   - 图片是否存在？
   - GitHub Pages 是否已启用？

3. **常见问题**
   - 网站显示 404：等待几分钟，清除缓存
   - 样式错误：检查 CSS 文件路径
   - 图片不显示：确认图片文件存在
   - 最后更新时间不显示：检查 GitHub 用户名是否正确
